## 快速开始

### 最小化配置

如果您只想快速测试此插件，请按以下步骤操作：

1. **在 WebUI 插件管理中启用此插件**

2. **打开插件配置，设置至少以下一项**：
   - `keywords` - 添加您想要检测的精确关键词
   - `alert_qqs` - 添加接收警告的 QQ 号

3. **发送测试消息**（包含配置的关键词）

4. **查看日志输出**，验证告警已触发

## 功能详解

### 1. 精确关键词匹配 (Exact Keyword Matching)

**说明**: 消息必须完全包含关键词（区分大小写）

**示例**:
- 关键词: `"骚扰"`
- ✅ 匹配: "你很骚扰"、"骚扰别人"
- ❌ 不匹配: "骚"、"骚扰"

**配置**:
```json
"keywords": ["骚扰", "辱骂", "敏感内容"]
```

### 2. 模糊关键词匹配 (Fuzzy Keyword Matching)

**说明**: 消息只需包含关键词的任何大小写变体

**示例**:
- 关键词: `"脏话"`
- ✅ 匹配: "说脏话"、"脏话文化"、"说脏话SB"、"脏话"
- ❌ 不匹配: "脏"、"话"

**配置**:
```json
"fuzzy_keywords": ["脏话", "不当内容"]
```

### 3. 事件停止功能 (Event Stopping)

当检测到关键词时，如果 `stop_chat` 设置为 `true`：
- ✅ 立即停止事件传播
- ✅ 机器人 **不会** 继续处理此消息
- ✅ 不会调用 AI 或其他插件处理
- ❌ 用户无法看到任何回复（包括警告）

**注意**: 这是在 AstrBot 的事件链中停止传播，不是发送"禁言"指令

## 配置指南

### 完整配置示例

```json
{
  "alert_qqs": [
    "123456789",
    "987654321"
  ],
  "keywords": [
    "骚扰别人",
    "不当言论",
    "禁用词汇"
  ],
  "fuzzy_keywords": [
    "脏话",
    "敏感话题",
    "不文明"
  ],
  "enabled": true,
  "stop_chat": true,
  "alert_message": "【⚠️ 关键词告警系统】\n\n📌 发送者信息:\n  名称: {sender}\n  QQ: {sender_id}\n\n📝 消息内容:\n  {message}\n\n🚨 触发关键词:\n  {keyword}\n\n⏰ 时间: $(date)\n\n✓ 已停止机器人回复"
}
```

### 配置项详解

#### alert_qqs
- **类型**: 字符串列表
- **说明**: 当检测到关键词时，将警告信息发送给这些 QQ
- **默认值**: `[]` (空列表)
- **示例**: `["123456789", "987654321"]`

#### keywords
- **类型**: 字符串列表  
- **说明**: 精确匹配关键词（完全包含才触发）
- **默认值**: `["敏感词1", "敏感词2"]`
- **大小写**: **区分大小写**
- **示例**: `["骚扰", "辱骂"]`

#### fuzzy_keywords
- **类型**: 字符串列表
- **说明**: 模糊匹配关键词（任何大小写变体都会触发）
- **默认值**: `[]` (空列表)
- **大小写**: **不区分大小写**
- **示例**: `["脏话", "敏感话题"]`

#### enabled
- **类型**: 布尔值
- **说明**: 是否启用关键词检测
- **默认值**: `true`
- **说明**: 设为 `false` 时，插件仍加载但不检测关键词

#### stop_chat
- **类型**: 布尔值
- **说明**: 检测到关键词时是否停止机器人回复
- **默认值**: `true`
- **说明**: 设为 `false` 时，仅发送警告，机器人继续正常回复

#### alert_message
- **类型**: 长文本
- **说明**: 警告消息的格式模板
- **支持的变量**:
  - `{sender}` - 发送者昵称
  - `{sender_id}` - 发送者 QQ 号
  - `{message}` - 原始消息内容
  - `{keyword}` - 触发的关键词
- **默认值**: `"【检测到关键词警告】\n发送者: {sender}\nQQ: {sender_id}\n消息内容: {message}\n检测词: {keyword}"`

## 充分示例

### 场景 1: 监控不文明用语

```json
{
  "alert_qqs": ["你的QQ号"],
  "keywords": [],
  "fuzzy_keywords": ["脏话", "人身攻击", "辱骂"],
  "enabled": true,
  "stop_chat": true,
  "alert_message": "【不文明用语告警】\n发送者: {sender} ({sender_id})\n内容: {message}\n触发词: {keyword}"
}
```

### 场景 2: 监控商业敏感词

```json
{
  "alert_qqs": ["运营QQ1", "运营QQ2"],
  "keywords": ["竞争对手名称", "私下交易", "价格战"],
  "fuzzy_keywords": [],
  "enabled": true,
  "stop_chat": false,
  "alert_message": "【商业敏感词告警】\n用户: {sender} - QQ: {sender_id}\n消息: {message}\n敏感词: {keyword}\n处理状态: 已发送警告，允许继续对话"
}
```

### 场景 3: 监控特定话题

```json
{
  "alert_qqs": ["管理员QQ"],
  "keywords": [],
  "fuzzy_keywords": ["涉政", "涉黄", "涉暴"],
  "enabled": true,
  "stop_chat": true,
  "alert_message": "🚨 严格内容检测\n用户: {sender_id}\n消息: {message}\n关键词: {keyword}\nAction: 已阻止消息处理"
}
```

## 故障排除

### 问题 1: 没有看到告警消息

**可能原因**:
1. QQ 号配置错误
2. 关键词没有正确配置
3. 插件未启用（`enabled: false`）

**解决方法**:
1. 检查配置中的 `alert_qqs` 是否填写了有效的 QQ 号
2. 运行 `/keyword_alert_status` 确认关键词已加载
3. 检查 AstrBot 日志，查看是否有错误信息

### 问题 2: 触发了不应该触发的消息

**可能原因**:
1. 关键词设置过于宽泛
2. 分不清精确匹配和模糊匹配

**解决方法**:
1. 使用更具体的关键词
2. 优先使用精确匹配 (`keywords`) 而非模糊匹配 (`fuzzy_keywords`)
3. 定期审查和更新关键词列表

### 问题 3: 插件加载失败

**可能原因**:
1. 配置文件格式错误
2. Python 环境问题

**解决方法**:
1. 检查 `_conf_schema.json` 是否是有效的 JSON 格式
2. 在 JSON 検證工具中验证配置
3. 查看 AstrBot 错误日志获取更多信息

## 技术细节

### 事件流

```
用户消息到达
    ↓
插件检测到消息事件
    ↓
比较关键词
    ├─ 精确匹配: 检查 keyword in message
    └─ 模糊匹配: 检查 keyword.lower() in message.lower()
    ↓
匹配到关键词?
    ├─ 是: 发送警告并考虑停止事件
    └─ 否: 继续正常流程
```

### 日志级别

插件使用以下日志级别：

| 级别 | 用途 | 示例 |
|------|------|------|
| INFO | 正常操作 | "插件已加载"、"配置已重载" |
| WARNING | 告警事件 | "检测到关键词"、"发送警告" |
| ERROR | 错误情况 | "发送失败"、"检测异常" |

### 性能考虑

- 关键词检查在每条消息的 Python 级别进行（低开销）
- 建议关键词数量不超过 100 个
- 模糊匹配比精确匹配稍慢（使用 `.lower()`）

### 安全考虑

- 所有日志输出都包含原始消息内容（请注意日志安全）
- 建议定期清理日志文件
- QQ 号会被记录在日志中（注意隐私）

## API 接口

### 命令列表

#### 1. `/keyword_alert_status` 或 `/告警状态`

查看插件当前状态

**响应示例**:
```
【关键词告警插件状态】
启用状态: ✓ 已启用
精确关键词数: 3
模糊关键词数: 2
接收警告QQ数: 2
停止聊天: 是

精确关键词: 骚扰, 辱骂, 不文明
模糊关键词: 脏话, 敏感词
```

#### 2. `/reload_alert_config` 或 `/重载告警配置`

重新加载配置（仅管理员可用）

**响应示例**:
```
✓ 告警配置已重新加载
```

## 常见配置错误

### ❌ 错误 1: JSON 格式问题

```json
{
  "keywords": ["脏话",],  // ❌ 多余的逗号
  "enabled": true
}
```

**正确方式**:
```json
{
  "keywords": ["脏话"],
  "enabled": true
}
```

### ❌ 错误 2: 数据类型错误

```json
{
  "alert_qqs": "123456789",  // ❌ 应该是数组
  "enabled": "true"  // ❌ 应该是布尔值
}
```

**正确方式**:
```json
{
  "alert_qqs": ["123456789"],
  "enabled": true
}
```
