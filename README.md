# 关键词告警插件 (Keyword Alert Plugin)

这是一个 AstrBot 插件，用于监听所有消息中的敏感关键词，当检测到关键词时立即向指定的 QQ 发送警告消息，并可选地停止机器人的正常聊天回复。

## 功能特性

✨ **关键词监控**
- 支持精确关键词匹配（完全相同）
- 支持模糊关键词匹配（包含即可触发）

🚨 **智能告警**
- 检测到关键词时立即发送警告消息
- 支持自定义警告消息模板
- 可配置接收警告的多个 QQ 号

⛔ **事件控制**
- 检测到关键词后可自动停止机器人聊天
- 支持热重载配置

📊 **管理命令**
- `/keyword_alert_status` 或 `/告警状态` - 查看插件运行状态
- `/reload_alert_config` 或 `/重载告警配置` - 重新加载配置（仅管理员）
- `/register_alert_origin` 或 `/注册告警` - 将当前 QQ 注册为告警接收者（需要私聊执行）
- `/unregister_alert_origin` 或 `/取消注册告警` - 取消当前 QQ 的告警接收注册

## 安装

1. 将插件文件放在 AstrBot 的 `data/plugins/` 目录下
2. 重启 AstrBot 或在 WebUI 插件管理中启用此插件

## 配置

在 AstrBot WebUI 的插件管理页面，找到"关键词告警插件"，点击配置按钮，配置以下内容：

### 配置项说明

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `alert_qqs` | 列表 | 需要被提醒的 QQ 号码列表 | `[]` |
| `keywords` | 列表 | 精确匹配关键词（完全相同才会触发） | `["敏感词1", "敏感词2"]` |
| `fuzzy_keywords` | 列表 | 模糊匹配关键词（包含这些字符串即可触发） | `[]` |
| `enabled` | 布尔值 | 是否启用关键词监控 | `true` |
| `stop_chat` | 布尔值 | 检测到关键词时是否停止机器人回复 | `true` |
| `stop_duration_seconds` | 整数 | 停止机器人聊天的时长（秒） | `300` |
| `alert_message` | 文本 | 警告消息的格式模板 | 详见下文 |

### 警告消息模板变量

使用以下变量来自定义警告消息：

- `{sender}` - 消息发送者昵称
- `{sender_id}` - 消息发送者 QQ 号
- `{message}` - 原始消息内容
- `{keyword}` - 触发的关键词

### 示例配置

```json
{
  "alert_qqs": ["123456789", "987654321"],
  "keywords": ["骚扰", "辱骂", "不文明"],
  "fuzzy_keywords": ["脏话", "敏感词"],
  "enabled": true,
  "stop_chat": true,
  "alert_message": "【关键词告警】\n✓ 发送者: {sender}\n✓ QQ: {sender_id}\n✓ 触发词: {keyword}\n✓ 消息: {message}"
}
```

## 使用示例

### 1. 查看插件状态

```
/keyword_alert_status
```

输出示例：
```
【关键词告警插件状态】
启用状态: ✓ 已启用
精确关键词数: 3
模糊关键词数: 2
接收警告QQ数: 2
停止聊天: 是

精确关键词: 骚扰, 辱骂, 不文明
模糊关键词: 脏话, 敏感词
```

### 2. 重载配置（管理员）

```
/reload_alert_config
```

### 3. 注册告警接收者

需要接收告警的 QQ 与机器人私聊，然后发送：
```
/register_alert_origin
```
或
```
/注册告警
```

注册成功后，插件会保存该 QQ 的私聊 origin，以便之后检测到关键词时直接发送私聊告警。

### 4. 管理停止聊天阻断

#### 查看当前阻断列表
```
/blocked_senders
```
或
```
/阻断列表
```

#### 解除指定用户阻断（管理员）
```
/unblock_sender 123456789
```
或
```
/解除阻断 123456789
```

#### 清除所有阻断记录（管理员）
```
/clear_blocked_senders
```
或
```
/清除阻断
```

这允许管理员在误判或手动结束阻断时，快速恢复用户的正常聊天能力。

### 4. 停止聊天时长

如果你希望机器人在检测到关键词后停止与用户继续对话，可以在插件配置中设置：
- `stop_chat`: 开启停止聊天
- `stop_duration_seconds`: 停止聊天的秒数

例如：
```json
{
  "stop_chat": true,
  "stop_duration_seconds": 600
}
```

这表示检测到关键词后，机器人将阻止该用户在接下来的 10 分钟内继续触发机器人聊天。

### 5. 工作流程

1. 用户发送消息：`"你太骚扰了"`
2. 插件检测到关键词 `"骚扰"`
3. 向配置的 QQ 号发送警告消息
4. 如果 `stop_chat` 为 `true`，停止机器人回复

## 日志输出

插件会在 AstrBot 日志中输出检测详情：

```

发送者: 小明 (123456789)

[WARNING] [关键词告警] 向QQ 987654321 发送警告: ...
[INFO] 已停止消息处理链，关键词: 骚扰
```

## 常见问题

### Q: 如何只启用精确匹配不启用模糊匹配？
A: 在配置中只填写 `keywords` 列表，保持 `fuzzy_keywords` 为空即可。

### Q: 警告消息如何真正发送给指定 QQ？
A: 当前插件支持两种告警发送方式：
1. 如果目标 QQ 已通过 `/register_alert_origin` 私聊机器人注册过 origin，插件会直接向该 QQ 私聊发送告警。
2. 如果目标 QQ 无 origin 注册，且当前消息来自群聊，插件会回退到当前会话发送告警消息，并尽量 @ 目标 QQ。
3. 如果目标 QQ 无 origin 注册，且当前消息来自私聊，插件不会将警告发送到当前用户，以避免继续回复该用户。

### Q: 如何临时禁用插件？
A: 在配置中将 `enabled` 改为 `false` 即可，无需卸载插件。

### Q: 关键词匹配是否区分大小写？
A: 精确匹配区分大小写，模糊匹配不区分大小写。
